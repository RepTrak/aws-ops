# AWS Ops

Point-in-time snapshot exporter and interactive architecture navigator for AWS infrastructure.

---

## Repository layout

```
aws-ops/
  export-infra-snapshot.sh   # snapshot export script
  NAVIGATOR_BLUEPRINT.md     # design document
  snapshots/
    latest.json              # points to the current snapshot folder
    2020-06-03T.../          # demo snapshot (included in repo)
    2020-06-02T.../          # older demo snapshot (for diff testing)
  navigator/                 # React/Vite architecture navigator app
```

Real snapshots generated by the export script are excluded from the repo via `.gitignore`. Only the two generic demo snapshots are committed so the navigator works immediately after cloning.

## Part 1 — Snapshot export

### Requirements

- AWS CLI v2
- `jq`
- `python3`
- An AWS profile with read-only access to the target account

### Basic usage

```bash
./export-infra-snapshot.sh
```

### Options

| Flag | Description |
|---|---|
| `--region <region>` | AWS region to export (default: CLI default region) |
| `--profile <profile>` | AWS profile to use |
| `--all-regions` | Export every enabled region sequentially |
| `--out-root <path>` | Override output directory (default: `snapshots/` next to the script) |
| `--with-secret-values` | Also export Secrets Manager and SSM Parameter values (off by default) |

```bash
# Examples
./export-infra-snapshot.sh --region us-east-1 --profile prod
./export-infra-snapshot.sh --all-regions --profile prod
```

### Region behaviour

- Without `--region`, the script uses the AWS CLI effective default (from env, profile, or config).
- Most services are **region-scoped**; each run produces one region's snapshot.
- IAM, Route53, S3, CloudFront, and Organizations are **global** and are captured once per run regardless of `--all-regions`.
- `--all-regions` captures every opted-in region sequentially and keeps the first region's snapshot as `latest.json` (it contains the global data).

### What gets exported

The script captures **metadata only** — no secret values by default. Coverage includes 50+ AWS service categories:

**Compute & containers**: ECS (clusters, services, tasks, task definitions, container instances), EC2 instances (ECS-tagged), Lambda, ECR

**Networking**: VPC, subnets, route tables, security groups, NACLs, internet/egress gateways, NAT gateways, VPC endpoints, transit gateways, VPN, Direct Connect, peering, Elastic IPs, Route53 (zones, records, health checks, traffic policies), Route53 Resolver, Cloud Map

**Load balancing & edge**: ALB/NLB (listeners, rules, target health, attributes), API Gateway (REST v1 + HTTP/WebSocket v2), CloudFront, WAFv2, ACM certificates, Shield

**Data**: RDS (with Proxy), Redshift (provisioned + Serverless), DocumentDB, DynamoDB, ElastiCache, MemoryDB, EFS, S3, OpenSearch

**Messaging**: SQS, SNS, EventBridge (buses, rules, targets, Scheduler), Kinesis, Firehose, MSK, Amazon MQ, Step Functions

**IAM & security**: IAM (roles, users, groups, policies, instance profiles, OIDC providers), KMS, Secrets Manager, SSM Parameters, CloudTrail, GuardDuty, Security Hub, Access Analyzer, Organizations/SCPs, Cognito

**CI/CD**: CodeBuild, CodePipeline, CodeDeploy, CodeStar Connections

**Observability**: CloudWatch (alarms, dashboards, anomaly detectors), CloudWatch Logs

### Output layout

```
snapshots/
  latest.json                         ← points to the most recent folder
  YYYY-MM-DDTHH-MM-SSZ-<region>/
    manifest.json                     ← region, timestamp, profile, flags
    raw/                              ← AWS API responses (JSON + NDJSON)
    derived/
      summary.json                    ← resource counts
      README.md                       ← human-readable summary
      service_topology.json           ← ECS services + containers + EFS + Cloud Map
      alb_to_service.json             ← ALB → target group → service (pre-joined)
      sg_connectivity.json            ← SG-to-SG inbound edge graph
      sg_members.json                 ← SG → member resources (inverse map)
      subnet_classification.json      ← public/private classification per subnet
      iam_role_resource_access.json   ← role → service_access map (policy analysis)
      iam_role_trust_analysis.json    ← role → trusted services/accounts
      cloudwatch_alarm_targets.json   ← alarm → monitored resource + actions
      secret_consumers.json           ← secret ARN → task definitions that use it
      kms_usage.json                  ← KMS key → encrypted resources
      pipeline_chains.json            ← CodePipeline source→build→deploy chain
      dynamodb_stream_consumers.json  ← DynamoDB stream → Lambda consumers
      apigw_auth_chain.json           ← API Gateway authorizer → Cognito pool
      sqs_dlq_chains.json             ← SQS queue → dead-letter queue
      task_eni_map.json               ← ECS task → ENI → IP/subnet/SGs
      vpc_endpoint_routes.json        ← VPC endpoint → subnets routed through it
      (+ several more topology files)
```

### Permissions

The script only makes read-only API calls (`list-*`, `describe-*`, `get-*`). A read-only IAM policy is sufficient. `--with-secret-values` additionally needs `secretsmanager:GetSecretValue` and `ssm:GetParameter`.

## Part 2 — Architecture Navigator

The navigator is a React application in the `navigator/` subdirectory. See [`navigator/README.md`](./navigator/README.md) for full usage.

### Quick start

```bash
cd navigator
npm install
npm run dev          # http://localhost:5173
```

The dev server automatically serves the parent `snapshots/` directory — no configuration needed. Two demo snapshots are included in the repo so the navigator works immediately after cloning.

### What it does

- **Empty canvas, intent-driven**: start blank, search for any resource to place it, expand its connections to explore the architecture
- **9 edge types**: Deployment, Data Flow, Network, IAM, Observability, Auth, Encryption, DNS, Logging — toggle any combination simultaneously
- **Snapshot diff**: select a second snapshot to compare; the canvas shows only added and modified resources with visual badges
- **AI handoff**: every node produces a copy-paste brief with all AWS identifiers an AI agent needs to generate correct CLI commands

---

## Notes

- Snapshots are **static and point-in-time** — they do not reflect live AWS state.
- `--with-secret-values` exports sensitive data; handle the output carefully.
- The navigator never makes live AWS API calls.
